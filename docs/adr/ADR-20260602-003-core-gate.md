# ADR-20260602-003: Core release gate

## 状态

Accepted

## 决策

Core gate 是发布前的本地证据协议，由 `release-check`、`release-final-check`、`release-preflight` 和 `score --min 9.8` 共同定义。

`release-check` 负责默认治理、测试、文档、contract、Evidence manifest 和 checksum 校验；`release-final-check` 单向调用 `context-release`、score gate 和 Evidence validation；`release-preflight` 必须带显式版本号；`score --min 9.8` 是结构评分门槛。

## 约束

- Core gate 必须在 `GOWORK=off` 下可复现。
- 必需 gate 不能被 skipped、planned 或 gap 结果伪装为 passed。
- `release/manifest/latest.json` 是生成 Evidence，不提交到源码历史。
- 非干净工作区不得声明 release ready。
- `context-release` 不得递归调用 `release-check` 或 `release-final-check`。

## 后果

变更 Makefile、harness、goalcli、release Evidence 或评分语义时，必须同步 `.agent/harness/harness.yaml`、`.agent/harness/gates.md`、`docs/standard/harness-gates.md`、`docs/standard/evidence-protocol.md` 和 traceability matrix。最终发布声明必须引用新鲜命令输出，不能只引用过期文档状态。
